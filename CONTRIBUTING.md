## üíª Making Code Contributions

We follow the **standard GitHub flow** for code contributions.

### 1. Prerequisites

* Read and agree to the terms of the **[LICENSE](LICENSE)**.
* Make sure your contribution aligns with the project‚Äôs goals. For major features or changes, consider discussing them in an issue first.

### 2. Setup

1.  **Fork** the repository.
2.  **Clone** your fork:
    ```bash
    git clone [https://github.com/](https://github.com/)<your-username>/<your-forked-repo>.git
    cd <your-forked-repo>
    ```
3.  Create a descriptive branch for your changes (e.g., `feature/add-logging`, `fix/typo-in-docs`):
    ```bash
    git checkout -b your-branch-name
    ```

### 3. Commit Guidelines

* Ensure your code follows existing style and formatting conventions.
* Write clear, concise commit messages. A good pattern is:
    ```text
    <Type>: <Short summary under 50 characters>
    ```
    **Examples:**
    * `Fix: Resolve memory leak in data parser`
    * `Feat: Add basic health check endpoint`

### 4. Submitting a Pull Request (PR)

1.  Push your branch to your fork:
    ```bash
    git push origin your-branch-name
    ```
2.  Open a new Pull Request against the **`main`** (or **`master`**) branch of the **mscsquared labs** repository.
3.  Fill out the PR template completely and reference any related issues (e.g., `Closes #123`).
